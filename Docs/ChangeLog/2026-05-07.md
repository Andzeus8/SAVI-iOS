## TestFlight Build 14 - Share Metadata Hotfix

- Fixed Release/TestFlight metadata regression where shared links could stay
  generic after save because Release had link metadata repair disabled.
- Re-enabled Release `LinkPresentation` metadata use on supported iOS versions
  and re-enabled stale metadata repair after app launch/foreground.
- Added device-aware metadata limits through `SaviPerformancePolicy`:
  - modern devices can repair more metadata concurrently;
  - iPhone 11/legacy devices repair only a tiny batch at launch/foreground.
- Prevented stale metadata repair from chaining through the whole library on
  older phones. Only share-import/deeplink/pasteboard saves can trigger one
  follow-up repair after their own metadata completes.
- Improved the share extension metadata window:
  - increased remote metadata timeout from `2.2s` to `3.8s`;
  - increased JSON/HTML request timeout from `2s` to `3.5s`;
  - enabled LinkPresentation fallback for the share extension on iOS 18+.
