# SAVI Privacy Policy Draft

Last updated: May 9, 2026

This is a publication-ready draft for the first SAVI beta. Review it against
the exact submitted build, public domain, company/legal details, and App Store
privacy labels before publishing.

## What SAVI Does

SAVI helps you save links, screenshots, notes, files, PDFs, audio, places, and
other things you want to find later.

## Data You Save

SAVI may store the content you choose to save, including titles, notes, tags,
folders, thumbnails, images, PDFs, screenshots, files, and links.

For the current private-save release, SAVI stores your library locally on your
device unless a feature clearly says it will use an external service.

## Metadata Fetching

When you save a web link, SAVI may contact the linked website or metadata
provider to fetch a title, description, source name, and thumbnail. This helps
the saved card look useful.

## Private Vault

Private Vault content is intended to stay private. SAVI must not publish Private
Vault content, send it to social features, or include it in analytics.

## Analytics

If analytics are enabled in a future release, SAVI will use a limited event
allowlist to understand app health and product usage. SAVI should not collect
private note text, document contents, screenshots, Private Vault content,
keystrokes, raw clipboard text, contacts, or private file names.

## Accounts And Sync

Future account and sync features may use Sign in with Apple and Supabase. Account
features should be optional unless a feature requires identity, such as public
link sharing or following friends.

## Public Social Features

Future social features are planned for public web links only. SAVI should not
allow public sharing of private files, PDFs, screenshots, Private Vault content,
or private documents.

## Data Deletion

If accounts are enabled, users must be able to request or perform account and
data deletion. See `/data-deletion`.
